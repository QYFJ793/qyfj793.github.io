Rust 模式匹配

一、模式匹配简介

模式匹配（Pattern Matching）是 Rust 中一种强大的语言特性，它允许我们根据不同的模式来执行不同的代码。它通常用于处理枚举类型、元组、结构体等数据结构，可以方便地对数据进行解构和匹配。

三、匹配枚举类型

枚举（Enum）是 Rust 中一种用于定义可变数据类型的方式。模式匹配与枚举结合使用可以方便地处理不同的枚举变体。

示例：匹配枚举

：IpAddrKind 是一个枚举类型，它有两个变体：V4 和 V6。V4 变体包含四个 u8 类型的字段，表示 IPv4 地址的四个部分；V6 变体包含一个 String 类型的字段，表示 IPv6 地址。在 match 表达式中，我们通过模式匹配来解构枚举变体，并提取其中的字段值。例如，当匹配到 IpAddrKind::V4(a, b, c, d) 时，a、b、c 和 d 分别被绑定到 V4 变体的四个字段值上，然后可以使用这些绑定的变量来构造 IPv4 地址的字符串。

四、匹配元组

元组（Tuple）是一种可以包含多个不同类型值的数据结构。模式匹配可以用于解构元组中的值。

示例：匹配元组

：point 是一个包含两个整数值的元组。在 match 表达式中，我们通过模式匹配来解构元组中的值。例如，(0, 0) 匹配元组的值为 (0, 0) 的情况；(x, 0) 匹配元组的第一个值为任意值（通过变量 x 绑定），第二个值为 0 的情况；(0, y) 匹配元组的第一个值为 0，第二个值为任意值（通过变量 y 绑定）的情况；(x, y) 匹配元组的两个值分别为任意值（通过变量 x 和 y 绑定）的情况。

五、匹配结构体

结构体（Struct）是一种自定义的数据类型，用于存储相关数据。模式匹配可以用于解构结构体中的字段。

示例：匹配结构体

：Point 是一个结构体，包含两个字段 x 和 y。在 match 表达式中，我们通过模式匹配来解构结构体中的字段。例如，Point { x, y: 0 } 匹配结构体的 y 字段值为 0，x 字段值为任意值（通过变量 x 绑定）的情况；Point { x: 0, y } 匹配结构体的 x 字段值为 0，y 字段值为任意值（通过变量 y 绑定）的情况；Point { x, y } 匹配结构体的两个字段分别为任意值（通过变量 x 和 y 绑定）的情况。

六、模式匹配的守卫

模式匹配的守卫（Guard）是一种额外的条件，用于在匹配模式时进行更精细的控制。只有当模式匹配成功且守卫条件为真时，才会执行对应的分支。

九、if let 语法

if let 是一种更简洁的模式匹配语法，用于处理只对一个模式感兴趣的情况。它类似于 match，但语法更简洁，且只处理一个分支。

输出结果

说明

  • if let 语法格式为 if let <pattern> = <expression> { ... }。
  • 如果 <expression> 匹配 <pattern>，则执行代码块。
  • 否则，可以使用 else 分支处理其他情况。
